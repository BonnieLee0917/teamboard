## Design Context

### Users
- Primary user: 小萌（老板），需要在手机或桌面上快速看到 5 个 AI agent 当前在做什么、哪里卡住、进度到哪。
- Secondary users: 团队内 5 个 agent（Bonnie / Kane / Haaland / Vivian / Rose），用于领任务、看状态、理解上下文。
- 使用场景：高频、短时查看；经常是在 Discord 沟通之后快速打开确认全局状态。需要 mobile friendly。

### Brand Personality
- 直接
- 清晰
- 高压下依然可靠
- 情绪目标：让老板感到可控、放心、一眼看懂；让执行者感到节奏明确、信息密度高但不混乱。

### Aesthetic Direction
- 产品形态：偏产品化的实时团队控制台，不做炫技，不做 hacker 炫光。
- 视觉方向：干净、紧凑、专业，带一点战情室/指挥台气质。
- 主题：浅色优先，适合白天高频查看与移动端阅读。
- 参考方向：Linear / GitHub Projects 的清晰信息架构 + 更强的状态可视化与团队感。
- Anti-reference：不要做成泛 AI dashboard；不要霓虹深色、玻璃拟态、大渐变、夸张动效。

### Design Principles
1. 一眼看全局：任何首屏都先回答“现在进度怎样、谁在忙、哪里卡住”。
2. 状态优先于装饰：颜色、动画、布局都为状态变化服务。
3. 紧凑但不压迫：信息密度高，但层级清楚、可扫读。
4. 移动端不阉割：手机上改为分视图，不隐藏核心信息。
5. 风险高亮克制但明确：blocked / failed 必须立刻可见，但不能把全局刷成报警界面。
